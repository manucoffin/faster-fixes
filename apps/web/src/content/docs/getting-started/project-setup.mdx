## Organizations

Every account belongs to an organization. Organizations are created during onboarding. You can invite team members with one of three roles:

- **Owner** — Full access, including billing and organization deletion
- **Admin** — Manage projects, members, and settings
- **Member** — View and manage feedback within assigned projects

## Creating a project

Each project requires a **name** and a **website URL**. The URL is used to validate feedback submissions through origin checking. Create one project per website.

## API key

An API key is generated when you create a project. The widget uses this key to authenticate with the Faster Fixes API.

- Only the last 4 characters are visible in the dashboard
- You can regenerate the key at any time — this immediately invalidates the previous key
- Store the key in an environment variable, never in source code

```bash title=".env.local"
NEXT_PUBLIC_FASTER_FIXES_API_KEY=ff_pk_...
```

## Widget configuration

Widget appearance is managed from **Project Settings** in the dashboard.

### Color

The widget accent color. Defaults to `#02527E`.

### Position

Where the floating button appears on the page:

| Value | Description |
| --- | --- |
| `bottom-right` | Bottom-right corner (default) |
| `bottom-left` | Bottom-left corner |
| `top-right` | Top-right corner |
| `top-left` | Top-left corner |
| `middle-right` | Middle of the right edge |
| `middle-left` | Middle of the left edge |

### Enabled / Disabled

Toggle the widget on or off without removing code from your application. When disabled, the widget script loads but renders nothing.
